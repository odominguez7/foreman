## How to fill this out

1. Pick a team. If you don't know which, ask the MAS.664 staff for a list of team repos. Choose one whose domain is far enough from Foreman that you'll learn something new (an agent framework, an evaluation tool, a different vertical).
2. Set a 90-minute timer. The point is honest first-time-user feedback, not a deep review.
3. Try to set up and run the project as if you'd never heard of it. Read the README cold. Don't DM the team.
4. Fill in this form as you go. Capture errors verbatim. Take screenshots if a UI is broken.
5. When you submit, send this form to the MAS.664 staff. The other team should not see your name.
